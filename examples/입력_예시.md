# LG U+ 5G 요금제 입력 예시

이 파일은 Graph Designer Agent를 테스트할 때 사용할 수 있는 입력 예시입니다.

---

## 입력 방법 1: 간단한 비즈니스 요구사항

```
LG U+ 5G 요금제 상담 챗봇을 위한 그래프 DB 설계:

엔티티:
- 요금제(Plan): 이름, 가격, 데이터 제공량, 음성 제공량
- 요금제 카테고리(PlanCategory): 5G 단말기, 5G 프리미어 등
- 혜택(Benefit): OTT 서비스, 데이터 추가 등
- 가입 조건(Condition): 나이 제한, 약정 기간 등

관계:
- 요금제는 카테고리에 속함 (BELONGS_TO)
- 요금제는 여러 혜택을 포함 (INCLUDES)
- 요금제는 가입 조건을 요구 (REQUIRES)
```

---

## 입력 방법 2: 상세한 요금제 정보

```
[내부용] LG 유플러스 5G 요금제
개정일: 2024년 5월 22일

대상: 5G 단말기 이용 고객

### 1. 5G 시그니처
- 월 이용료: 130,000원
- 데이터: 무제한
- 공유 데이터: 60GB + 60GB (기본 + 추가)
- 음성/문자: 무제한
- OTT 팩: 2개 선택 가능 (넷플릭스, 디즈니+, 웨이브 등)
- 스마트기기: 2회선 무료 (워치, 태블릿 등)
- 로밍: 50% 할인
- 선택약정 할인: 월정액의 25% (다이렉트 요금제 제외)
- 카테고리: 5G 단말기

### 2. 5G 프리미어 슈퍼
- 월 이용료: 115,000원
- 데이터: 무제한
- 공유 데이터: 50GB + 50GB
- 음성/문자: 무제한
- OTT 팩: 선택 가능
- 선택약정 할인: 월정액의 25%
- 카테고리: 5G 프리미어

### 3. 5G 프리미어 에센셜
- 월 이용료: 95,000원
- 데이터: 40GB
- 음성/문자: 무제한
- 테더링/쉐어링: 10GB
- 선택약정 할인: 월정액의 25%
- 카테고리: 5G 프리미어

### 4. 5G 스탠다드
- 월 이용료: 75,000원
- 데이터: 30GB
- 음성/문자: 무제한
- 테더링/쉐어링: 5GB
- 카테고리: 5G 프리미어

### 특별 혜택 및 조건

#### 나이 할인
- 대상: 만 34세 이하
- 혜택: 추가 할인 (요금제별 상이)
- 조건: 신분증 확인 필요

#### 가족 결합 할인
- 대상: 2회선 이상 가족 결합
- 혜택: 데이터 2배 제공
- 조건: 가족 관계 증명 필요

#### 데이터 쉐어링
- 대상: 시그니처/프리미어 플러스 이상
- 혜택: 보조기기 2회선 무료
- 조건: 주 회선과 동일 명의

#### 선택약정 할인
- 대상: 모든 요금제 (다이렉트 제외)
- 혜택: 월정액의 25% 할인
- 조건: 24개월 약정
```

## 입력 방법 3: 자연어 요구사항 (추가 기능)

```
다음 기능들을 그래프 스키마에 추가해주세요:

1. "LG U+ 5G 요금제 중 만 34세 이하 할인 혜택 구조를 설계해줘."
   → 나이 조건과 할인 혜택 간의 관계 추가

2. "가족 결합 시 데이터 2배 제공 조건도 포함해."
   → 가족 결합 조건과 데이터 보너스 관계 추가

3. "데이터 쉐어링: 시그니처/프리미어 플러스 이상은 보조기기 2회선 무료 혜택 연결."
   → 요금제 등급과 쉐어링 혜택 간의 계층 구조 추가

4. "OTT 서비스별로 어떤 요금제에서 선택 가능한지 관계를 명확히 해줘."
   → OTT 서비스와 요금제 간의 다대다 관계 추가
```

## 예상 출력 (Agent가 생성해야 할 결과)

### 1. 그래프 스키마 구조
```
Nodes:
- Plan (id, name, price, data_limit, voice_limit, shared_data)
- PlanCategory (id, category_name, description)
- Benefit (id, benefit_type, description, value)
- Condition (id, condition_type, value, description)
- OTTService (id, service_name, provider)
- AgeDiscount (id, min_age, max_age, discount_rate)

Edges:
- Plan -[BELONGS_TO]-> PlanCategory
- Plan -[INCLUDES]-> Benefit
- Plan -[REQUIRES]-> Condition
- Plan -[OFFERS]-> OTTService
- Plan -[ELIGIBLE_FOR]-> AgeDiscount
```

### 2. Spanner DDL
```sql
CREATE TABLE Plan (
  id STRING(36) NOT NULL,
  name STRING(100),
  price INT64,
  data_limit INT64,
  voice_limit INT64,
  shared_data STRING(50),
) PRIMARY KEY (id);

```

## 사용 방법

### Agent 테스트 시:

1. **간단한 테스트**: "입력 방법 1"의 내용을 복사하여 Agent에 입력
2. **상세한 테스트**: "입력 방법 2"의 전체 요금제 정보를 입력
3. **반복 개선**: "입력 방법 3"의 자연어 요구사항으로 스키마 수정
4. **구조화된 데이터**: "입력 방법 4"의 표 데이터를 파일로 첨부

### 검증 포인트:

✅ Agent가 모든 엔티티를 정확히 추출했는가?
✅ 관계가 올바르게 정의되었는가?
✅ DDL이 Spanner Graph 문법을 준수하는가?
✅ 시각화가 명확하게 표현되었는가?
✅ 특별 혜택 및 조건이 스키마에 반영되었는가?
