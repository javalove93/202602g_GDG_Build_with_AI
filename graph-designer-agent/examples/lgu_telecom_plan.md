## 그래프 DB 설계를 위한 엔티티 및 관계

### 엔티티 (Nodes)
1. **Plan** (요금제)
   - id, name, price, data_limit, voice_limit, sharing_data

2. **PlanCategory** (요금제 카테고리)
   - id, category_name, description

3. **Benefit** (혜택)
   - id, benefit_type, description, value
   - 예: OTT 팩, 스마트기기, 로밍 할인

4. **Condition** (가입 조건)
   - id, condition_type, value, description
   - 예: 나이 제한, 신용 등급, 약정 기간

5. **Discount** (할인)
   - id, discount_type, discount_rate, description
   - 예: 선택약정 할인, 만 34세 이하 할인, 가족 결합 할인

### 관계 (Edges)
1. **BELONGS_TO**: Plan → PlanCategory
   - 요금제가 어느 카테고리에 속하는지

2. **INCLUDES**: Plan → Benefit
   - 요금제가 어떤 혜택을 포함하는지

3. **REQUIRES**: Plan → Condition
   - 요금제 가입에 필요한 조건

4. **OFFERS**: Plan → Discount
   - 요금제가 제공하는 할인

5. **ELIGIBLE_FOR**: Condition → Discount
   - 특정 조건을 만족하면 받을 수 있는 할인
